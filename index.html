BookStore     WebService

Mini documentation qui pésente comment utiliser se service web.

Récupération des données:

Pour la récupération des données provenant de la base de données de l'application, on doit uitilisé une adresse URL, puis transmetre les variables par la methode de transfère de données POST.


Le resultat des requêtes sont présentés sous forme d'objet JSON => {"success": bool, "value": mixed}

En cas d'erreurs, elles sont présentés sous forme d'objet JSON => {"error": bool, "message": string}

Descriptions des différents attributs :
  • success : Etat de la requête, pour savoir si elle s'est exécutée.
  • value : Valeur qui contient le resultat de la requête.
  • error : Etat d'erreur dans la requête, pour savoir si elle s'est exécutée avec erreur.
  • message : Message qui donne le détail sur l'erreur produite dans la requête.
Tableau contenant les différent options de récupérations
Chemein de récupération (adresse url)Variable(s) à passer
/bookstore/get/book.php
  • $_POST['class'] - pour récuperer tous les livres d'une classe.
  • $_POST['cycle'] - pour récuperer tous les livres d'un cycle.
  • $_POST['search']:(recherche) - pour récuperer tous les livres dont le titre, l'édition, l'auteur correspond au contenu de search
/bookstore/get/classe.php
  • Pour récuperer tous les classes, on ne passe pas de variable. Tout
  • $_POST['cycle'] - pour récuperer tous les classes d'un cycle.
/bookstore/get/commend.php
  • $_POST['id'] - pour récuperer une commande en fonction de son identifiant.
  • $_POST['user_id'] - pour récuperer tous les commandes d'un client avec l'id (user_id).
  • $_POST['user_id'], $_POST['book_id'] - pour savoir si un livre est commandé par un client.
  • $_POST['bill_ref']:(ref facture) - pour récuperer tous les commandes d'une facture avec la réference (bill_ref).
/bookstore/get/bill.php
  • $_POST['shipping_type']:(type livraison) - Pour récuperer tous les factures d'un type de livraison (Insatantaneous, Express, Standard)
  • $_POST['shipping_state']:(état livraison) - Pour récuperer tous les factures pour un état (In Course, Delivered, )
/bookstore/get/shippingAddress.php
  • $_POST['has_sa']:(id utilisateur) - Pour vérifier si un utilisateur possède une adresse de livraison.
  • $_POST['user_id'] - Pour récuperer l'adresse de livraison d'un client.
  • $_POST['ship_ref']:(ref commande) - Pour récuperer une adresse de livraison en fonction de sa reference.
/bookstore/get/user.php
  • $_POST['user_id'] - Pour récuperer un utilisateur (client).
  • $_POST['login'], $_POST['pass'] - Pour l'authentification d'un utilisateur (client) à partir de son email.
  • $_POST['phone'], $_POST['pass'] - Pour l'authentification d'un utilisateur (client) à partir de son numero de téléphone.

Ajout de données dans la database :

Pour l'ajout d'informations dans la base de données de l'application, on procède de la même manière que pour la recupération de données (par une adresse URL et la methode POST).

Tableau contenant les différent options d'ajout
Chemein de récupération (adresse url)Variable(s) à passer
/bookstore/post/user.php
  • $_POST['name'], $_POST['surname'], $_POST['phone'], $_POST['login']:(email), $_POST['pass'] - pour ajouter un nouveau utilisateur (client).
/bookstore/post/commend.php
  • $_POST['book_id'], $_POST['quantity'], $_POST['user_id'] - pour ajouter une nouvelle commande.
/bookstore/post/bill.php
  • $_POST['user_id'], $_POST['shipping_add_ref']:(ref_adresse), $_POST['shipping_type']:(type livraison), $_POST['total_prise'] - pour ajouter une nouvelle factrue.
/bookstore/post/shippingAddress.php
  • $_POST['receiver_name'], $_POST['phone'], $_POST['district']:(ville), $_POST['street']:(localite), $_POST['more_desc']:(desc. adresse), $_POST['user_id'] - pour ajouter une nouvelle adresse de livraison.

Mise à jour de données dans la database :

Pour la mise à jour d'informations dans la base de données de l'application, une fois de plus on procède de la même manière que pour la recupération de données (par une adresse URL et la methode POST).

Tableau contenant les différent options de mise à jour
Chemein de récupération (adresse url)Variable(s) à passer
/bookstore/update/user.php
  • $_POST['login']:(email), $_POST['user_id'] - pour modifier l'adresse email d'utilisateur (client).
  • $_POST['phone'], $_POST['user_id'] - pour modifier le numero de téléphone d'utilisateur (client).
  • $_POST['pass'], $_POST['user_id'] - pour modifier le mot de passe d'utilisateur (client).
  • $_POST['is_admin']:(1/0), $_POST['user_id'] - pour pour indiquer si un tilisateur (client) est admin ou pas.
  • $_POST['last_connexion']:(id utilisateur) - pour modifier la date de la dernière connexion d'utilisateur (client).
/bookstore/update/commend.php
  • $_POST['validate']:(id commande) - pour valider une commande.
  • $_POST['billed']:(id commande), $_POST['bill_ref']:( ref facture) - pour facturer une commande.
  • $_POST['new_q']:(quantité livre), $_POST['cmd_id'], $_POST['book_prise']:(prix livre) - pour modifier la quantité de livre.
/bookstore/update/bill.php
  • $_POST['new_state']:(état), $_POST['bill_ref']: (ref facture) - pour modifier l'état d'une factrue (Delivered, In Course, Obsolete).
  • $_POST['new_tp']:(prix total), $_POST['bill_ref']: (ref facture) - pour modifier le prix total d'une factrue.
  • $_POST['ship_type']:(type livraison), $_POST['bill_ref']: (ref facture) - pour modifier le type de livraison d'une factrue.
/bookstore/update/shippingAddress.php
  • $_POST['username'], $_POST['shipping_ref']:(ref adresse de livraison) - pour modifier le nom du receleur dans une adresse de livraison.
  • $_POST['phone'], $_POST['shipping_ref']:(ref adresse de livraison) - pour modifier le numero de téléphone du receleur dans une adresse de livraison.
  • $_POST['district']:(ville), $_POST['shipping_ref']:(ref adresse de livraison) - pour modifier la ville d'une adresse de livraison.
  • $_POST['street']:(localité), $_POST['shipping_ref']:(ref adresse de livraison) - pour modifier la localité d'une adresse de livraison.
  • $_POST['add_desc']:(desc. adresse), $_POST['shipping_ref']:(ref adresse de livraison) - pour modifier la description d'une adresse de livraison.

Suppréssion de données dans la database :

Pour la suppréssion d'informations dans la base de données de l'application, une fois de plus on procède de la même manière que pour la recupération de données (par une adresse URL et la methode POST).

Tableau contenant les différent options de suppréssion
Chemein de récupération (adresse url)Variable(s) à passer
/bookstore/delete/commend.php
  • $_POST['cmd_id']:(id commande) - pour supprimer une commande.
/bookstore/delete/shippingAddress.php
  • $_POST['ship_add_ref']:(ref adresse livraison) - pour supprimer une adresse de livraison.
/bookstore/delete/bill.php
  • $_POST['bill_ref']: (ref facture) - pour la suppréssion d'une factrue.
